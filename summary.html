  1. Example →

    Let's start off with an empty HTML document.

    • Create an index.html file.
    • Add head and body tags.
    • Add meta tags, title, and some dummy content.
  2. Example →

    Now, we need to start putting the pieces of our app in place.

    • Flesh out the HTML structure of our app.
    • Use a header and h1 for a top bar, and an ol with buttons and imgs for questions.
    • Extra divs are included to be used for layout later.
  3. Example →

    We can use Bootstrap to make our app look good fast.

    • Discuss use of libraries.
    • Download bootstrap from getbootstrap.com and link it to index.html.
    • Alternatively, use the CDN: http://netdna.bootstrapcdn.com/bootstrap/3.0.0-rc2/css/bootstrap.min.css.
  4. Example →

    Bootstrap styles are applied using classes.

    • The header gets class "navbar navbar-fixed-top". The ol gets class "list-inline".
    • The buttons get class "btn btn-*" where * is one of default, info etc. listed here. Divs get class "container".
    • Use col-* classes for responsive layout
  5. Example →

    Tweaking Bootstrap with custom CSS.

    • Create a css file and link it to index.html.
    • The body needs padding-top: 70px so the top bar doesn't cover the questions.
    • The h1 needs margin: 0 so the top bar has a normal size.
  6. Example →

    We can use jQuery to make adding interaction to our app easier.

    • Include jQuery from Google's CDN: http://ajax.googleapis.com/ajax/libs/jquery/1.10.2/jquery.min.js.
    • Create an empty .js file and link it to the document.
    • Intro to JS: a jQuery "ready" event that triggers an alert.
  7. Example →

    Storing and retrieving our questions.

    • Questions are stored in a file called questions.json.
    • For each question, answer is the URL to the correct answer, and theme is the query that will be used to get the wrong answers.
    • Modify the JS to use $.get to load the questions. Console.log the result, introducing the devtools.
  8. Example →

    Getting our questions into the document.

    • Include handlebars.js in the document. handlebarsjs.com.
    • Get rid of dummy questions. Move one into a script type="text/html" tag, and add Handlebars bits for looping and conditionals
    • Compile the template in JS. Fill it with data, and add it to the DOM.
  9. Example →

    Using the Flickr API.

    • Get a Flickr API key from the flickr website: http://www.flickr.com/services/apps/create/apply/
    • Add the key and secret as variables in the JS file.
    • Make a function that uses the photo search endpoint: flickr.photos.search and console.log the results.
  10. Example →

    Loading photos from Flickr.

    • Modify the loadPhotos function so that it takes a callback, and passes it an array of objects with "url" and "correct" properties.
    • Use this new data in the template.
    • Google for "shuffle javascript array". Use the first result to randomise the answers.
